    Install Docker Engine Edge Release

    For beta installation and the easiest way which works across all Linux flavours. See Docker documentation.

    1
2
    $ curl -fsSL https://get.docker.com -o get-docker.sh
$ sudo sh get-docker.sh

    Follow the script which asks you to add user into docker group so you don't have to use sudo all the time. (Remember to log out after)

    Confirm install:

    1
    sudo docker version

    Running A Container

    1
    docker container run --publish 80:80 nginx

    This command above does the following:

    1. Downloaded image 'nginx' from Docker Hub
    2. Started a new container from that image
    3. Opened port 80 on the host IP
    4. Routes that traffic to the container IP, port 80.

    You'll get blind error if left number (host port) is being used by anything else. You can change the port to anything else that's free ie 6753:80.

    The above command will run the container in the foreground. To run it in the background use --detach.

    1
    docker container run --publish 80:80 --detach nginx

    To see what containers are running use the new command (replaced docker ps)

    1
    docker container ls

    To stop the container

    1
    Docker container stop <container ID/names>

    To show all containers running or not

    1
    docker container ls -a

    If you don't give a container a name it will auto generate them.

    1
2
3
    ajoyner@docker:~$ docker container ls
CONTAINER ID        IMAGE               COMMAND                  CREATED             STATUS              PORTS                NAMES
0892156fbdd8        nginx               "/docker-entrypoint.…"   4 minutes ago       Up 4 minutes        0.0.0.0:80->80/tcp   relaxed_khorana

    Interestingly the names come from an open sourced list with <adjectives>_<notable scientists/hackers>.

    To name a container use --name tag

    1
    docker container run --publish 80:80 --detach --name webhost nginx

    When running containers in the background you can't see the logs. To display logs do

    1
    docker container logs <name/container ID>

    To see the processes running in a container

    1
    docker container top <name/container ID>

    To remove a container

    1
    docker container rm 97 08 2d

    Above is three containers and just the first unique parts of the container ID

    To force remove a container If a container is running and you can be arsed to stop it use the -f tag to force it to stop and remove.

    1
    docker container rm -f 97

    Shell Inside Containers

    Start new container interactively

    1
    docker container run -it

    Run additional command in existing container

    1
    docker container exec -it

    So the -it option is two options together. -t creates a pseudo shell & -i makes it interactive.
    You can also run a bash shell inside a container. 

    1
2
    ajoyner@docker:~$ docker container run -it --name proxy nginx bash
root@eabc41e54023:/#


    When you exit out of a container, specially an OS image it will stop. To start it again you don't run but start it.

    1
    docker container start -ai centOS

    We still want to be interactive but this time we use the attached -a option.


    To run a command in a running container you use the exec option.

    1
    docker container exec -it mysql bash

    This will run a bash shell inside a running mysql server.

    Docker Networking

    To check what ports are mapped to a certain container

    1
2
    ajoyner@docker:~$ docker container port webserver 
80/tcp -> 0.0.0.0:80

    Check IP address of a specific container

    1
2
    ajoyner@docker:~$ docker container inspect --format '{{ .NetworkSettings.IPAddress }}' webserver 
172.17.0.3

    This is a NAT address as the host is on the 192.168.0.0/24 network.
